Steps to set up GITHUB

  1. Log in to Github
  2. Go to www.github/new
  3. Follow the prompts and enter in details
  4. Copy the https github link
  5. Open terminal and go to the directory you want the clone project in
  6. To clone the git to your local system - type in %git clone <git@github.com:roycesc/name_of_git.git>
  7. To add a file to the git repository type in to terminal - %git add <filename>
  8. To commit the changes type git commit -m "message"
  9. Final step - push the local git to the github online repository, in the terminal
    • Run % git status
    • Run % git push

Add & Commit

If you want to add all the files and commit at the same time - %git commit -am "comment"- which means commit all the files that have changes and a message. The push it to the github repository %git push

Pull form github - Get the latest version from the repository

%git pull

Merge Conflict

If there are conflicts, git will alert to these issues. Conflict happen on a line by line item, if a single line of code has been edited by git will alert the programmer to the difference to each line and will require the programmer to make the adjustments.

To remove these issues you need to;

  1. Remove all the merge conflict markers from the file
  2. Modify the file until the programmer is happy with it
  3. The do a % git commit -am "Fix the merge issues"
  4. % git push

Useful git commands

Git Command Description
% git log Useful to read all the commits made
% git reset To go back to an earlier commit
% git reset --hard <hash commit> Type in the has from git log to roll back to that version
% git reset --hard origin/master Reverts back to the last commit on the git hub repository

Branching

Branching gives you the ability to work on a feature without disturbing the master version of the code
